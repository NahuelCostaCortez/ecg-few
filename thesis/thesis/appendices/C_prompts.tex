\chapter{Mensajes y respuestas estructuradas}
\label{app:prompts}

\section{Criterio de conservación}

Los mensajes forman parte del protocolo experimental. En un modelo de visión y lenguaje, cambios pequeños en instrucciones, orden o formato pueden modificar la respuesta. Por esa razón se conserva el contenido de los prompts y el esquema de salida exigido.

\section{Sistema para tarea morfológica}

\begin{lstlisting}
You are evaluating single-beat right precordial ECG images.
Return only the requested JSON. Do not include prose, markdown, caveats, a
diagnosis, or a clinical recommendation.
\end{lstlisting}

\section{Tarea morfológica}

\begin{lstlisting}
Analyze this single-beat right precordial ECG image from lead {lead}.

Evaluate these morphology findings independently:

RBBB:
  - terminal RSR' or rSr' morphology in the QRS complex
  - delayed terminal right-precordial activation compatible with right bundle branch block

ST_ELEVATION:
  - J-point or ST segment remains elevated above the baseline after the QRS complex
  - coved, convex, or saddleback elevation can count when visually present

T_WAVE_INVERSION:
  - the T wave after the ST segment is predominantly negative below baseline

Return only valid JSON with boolean values for exactly these keys:

{
  "RBBB": true,
  "ST_ELEVATION": true,
  "T_WAVE_INVERSION": true
}
\end{lstlisting}

\section{Sistema para tarea clínica}

\begin{lstlisting}
You are evaluating right precordial ECG beat images from one patient.
Return only the requested JSON. Do not include prose, markdown, caveats,
a diagnosis, or a clinical recommendation.

The label is a research reference for suspected Brugada triage. It must not be
used as an autonomous diagnosis.
\end{lstlisting}

\section{Tarea clínica directa}

\begin{lstlisting}
Analyze this single right precordial ECG beat image from one patient.

Decide whether the patient is clinically compatible with the Brugada-positive
reference used in this study.

Return only valid JSON with exactly this key:

{
  "clinical_brugada": true
}
\end{lstlisting}

\section{Construcción del contexto}

En zero-shot no se añaden demostraciones. El modelo recibe el mensaje de sistema, la tarea y la imagen objetivo. En ICL normal, cada demostración contiene una imagen y la respuesta JSON correcta, sin forzar explícitamente la proporción de clases. En ICL balanceado, la selección intenta incluir pacientes positivos y negativos siempre que el pliegue lo permita. En ambos casos, el contexto nunca incluye el paciente reservado para test.

Las respuestas se validan de forma estricta. Una salida con claves ausentes, claves adicionales, texto libre o tipos incorrectos se registra como fallo de formato. En la evaluación final, la tasa de JSON inválido fue 0, por lo que las diferencias de rendimiento proceden de las decisiones del modelo y no de errores sintácticos.
